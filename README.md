# Project Structure and Execution Instructions

## Project Directory Structure

Smart-healthcare2.0/

├── angular_frontend/

├── django_backend/

├── pyspark_etl/

├── datasets/

├── mysql_scripts/

└── docs/

---

# 1. angular_frontend/

Purpose:
The Angular frontend provides the user interface for the healthcare management system.

Modules:

* Login
* Dashboard
* Patients
* Doctors
* Appointments
* Billing
* Treatments

Technologies:

* Angular 20
* Bootstrap
* TypeScript
* Angular Router
* HTTP Client

Execution:

```bash
cd angular_frontend/smartcare-ui

npm install

ng serve
```

Application URL:

```text
http://localhost:4200
```

---

# 2. django_backend/

Purpose:
The Django backend provides REST APIs and authentication services.

Features:

* REST APIs
* JWT Authentication
* Swagger Documentation
* Database Connectivity

Technologies:

* Django
* Django REST Framework
* Simple JWT
* CORS Headers
* drf-yasg

Execution:

```bash
cd django_backend

source ../venv/bin/activate

python manage.py runserver
```

Backend URL:

```text
http://127.0.0.1:8000
```

Swagger Documentation:

```text
http://127.0.0.1:8000/swagger/
```

---

# 3. pyspark_etl/

Purpose:
This module performs ETL operations on raw healthcare datasets.

Responsibilities:

* Read CSV files.
* Clean data.
* Transform data.
* Generate processed files.

ETL Jobs:

* patient_etl.py
* doctor_etl.py
* appointment_etl.py
* billing_etl.py
* treatment_etl.py

Execution:

```bash
python pyspark_etl/jobs/patient_etl.py

python pyspark_etl/jobs/doctor_etl.py

python pyspark_etl/jobs/appointment_etl.py

python pyspark_etl/jobs/billing_etl.py

python pyspark_etl/jobs/treatment_etl.py
```

Requirements:

* Java 17
* PySpark

---

# 4. datasets/

Purpose:
Stores both raw and processed healthcare datasets.

Structure:

datasets/

├── raw/

└── processed/

Raw Files:

* patients.csv
* doctors.csv
* appointments.csv
* billing.csv
* treatments.csv

Processed files are generated by ETL jobs.

---

# 5. mysql_scripts/

Purpose:
Contains database creation scripts and SQL statements.

Functions:

* Create database.
* Create tables.
* Verify records.
* Perform testing queries.

Example:

```sql
CREATE DATABASE smartcare;

USE smartcare;
```

Verification:

```sql
SELECT COUNT(*) FROM patients;

SELECT COUNT(*) FROM doctors;
```

# Project Startup Sequence

Step 1:

Run MariaDB.

Step 2:

Run ETL jobs.

Step 3:

Load data into MariaDB.

Step 4:

Start Django server.

```bash
python manage.py runserver
```

Step 5:

Start Angular application.

```bash
ng serve
```
